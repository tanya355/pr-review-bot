# AI-Powered PR Review Assistant

A GitHub Action that automatically reviews pull requests using OpenAI. When a PR is opened or updated, the action fetches the diff, evaluates it against a configurable set of criteria, and posts structured feedback as a PR comment.

---

## How it works

1. A pull request is opened or updated
2. GitHub Actions triggers the workflow
3. The action fetches the PR diff via the GitHub REST API
4. The diff is combined with your `review-config.yml` and sent to OpenAI
5. The response is parsed and posted as a structured review comment on the PR

---

## Architecture

```
PR opened/updated
      │
      ▼
GitHub Actions workflow (Node.js runner)
      │
      ▼
Fetch PR diff (GitHub REST API)  ◄──── review-config.yml
      │
      ▼
OpenAI API call (diff + rules as prompt)
      │
      ▼
Post structured review comment on PR
```

## Getting started

### Prerequisites

- A GitHub repository
- An OpenAI API key

### Installation

1. Clone this repository or copy the workflow file into your project:

```bash
git clone https://github.com/your-username/ai-pr-review-assistant.git
```

2. Add your OpenAI API key as a GitHub repository secret:

```
Settings → Secrets and variables → Actions → New repository secret
Name: OPENAI_API_KEY
```

3. Copy the workflow file to your target repo:

```bash
cp .github/workflows/pr-review.yml your-project/.github/workflows/
```

4. Copy and customise `review-config.yml` to define your team's review criteria.

## WORKFLOW
 
1. Load and validate environment variables (config.ts)
2. Initialize GitHub client (github.ts)
3. Initialize file filter from review-config.yml (fileFilter.ts)
4. Initialize OpenAI reviewer (openai.ts)
5. Fetch PR files from GitHub
6. Filter files based on ignore patterns
7. Generate diff string
8. Call OpenAI API to review code
9. Handle quota errors gracefully
10. Post review comment back to GitHub
11. Exit with appropriate status code
 
QUOTA EXCEEDED HANDLING
 
If OpenAI quota is exceeded, the workflow will:
- Catch the quota error
- Log "QUOTA_EXCEEDED: [message]" to stderr
- Include a link to the billing page
- Exit with code 1 (failure)
- Leave the PR unreviewed (no comment posted)

## Why I built this

At KPMG, I manage a team of 11 engineers across multiple streams. Code review is one of the highest-leverage activities for maintaining quality — but it's also one of the most time-consuming. This tool acts as a first-pass reviewer that flags common issues before the PR even reaches a human reviewer, freeing up senior engineers to focus on architecture and logic rather than style and hygiene.

The config-driven approach was intentional: different teams have different standards, and I wanted the review criteria to be owned by the team in version control, not hardcoded in the tool.
